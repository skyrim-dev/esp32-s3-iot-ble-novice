# 应用层 (Application Layer)

## 层级说明

应用层是嵌入式系统的最顶层，负责：
- **系统入口** - 程序启动入口点
- **业务逻辑** - 核心功能实现
- **模块协调** - 调用下层组件完成功能
- **任务调度** - FreeRTOS 任务管理

## 目录结构

```
main/
├── main.c          # 应用入口 (app_main)
├── CMakeLists.txt  # 组件配置
└── idf_component.yml  # IDF 组件清单
```

## 设计原则

| 原则 | 说明 |
|------|------|
| 简洁性 | 只负责协调，不实现细节 |
| 依赖注入 | 通过组件依赖获取功能 |
| 单向依赖 | 只依赖下层组件，不被依赖 |

## 架构位置

```
┌────────────────────────────────────────────────────┐
│                  应用层 (Application)              │
├────────────────────────────────────────────────────┤
│  职责:                                             │
│  • 业务逻辑编排                                    │
│  • 任务创建和调度                                  │
│  • 模块初始化顺序控制                              │
│  • 用户功能入口                                    │
├────────────────────────────────────────────────────┤
│  依赖: 网络层、协议层、驱动层                      │
│  被依赖: 无                                        │
└────────────────────────────────────────────────────┘
```
