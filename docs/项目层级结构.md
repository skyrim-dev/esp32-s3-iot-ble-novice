# ESP32 项目目录结构规范

## 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                        应用层 (app/main)                    │
│                   业务逻辑实现、主程序入口                     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       协议层 (protocol)                     │
│            通信协议实现、数据格式解析与封装                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       网络层 (network)                       │
│              网络连接管理、通信接口封装                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       驱动层 (driver)                       │
│              硬件外设驱动、传感器/执行器驱动                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                        ESP-IDF HAL                          │
│                    硬件抽象层、底层驱动                        │
└─────────────────────────────────────────────────────────────┘
```

## 层级说明

### 1. 应用层 (app/main)
**职责**：
- 业务逻辑实现
- 主程序入口 `app_main()`
- 功能模块调度
- 用户交互逻辑

**文件命名**：
- `app_main.c` - 主程序
- `business_xxx.c` - 业务模块

### 2. 协议层 (protocol)
**职责**：
- 通信协议实现（MQTT、HTTP、CoAP、WebSocket 等）
- 数据格式解析与封装（JSON、Protobuf、XML 等）
- 协议编解码
- 心跳/保活机制

**文件命名**：
- `mqtt_client.c/h` - MQTT 协议
- `http_client.c/h` - HTTP 协议
- `json_parser.c/h` - JSON 解析

### 3. 网络层 (network)
**职责**：
- 网络连接管理
- WiFi/以太网/蓝牙控制
- 网络状态监控
- 自动重连机制
- IP 地址获取

**文件命名**：
- `wifi_manager.c/h` - WiFi 管理
- `eth_manager.c/h` - 以太网管理
- `net_config.c/h` - 网络配置

### 4. 驱动层 (driver)
**职责**：
- 硬件外设驱动
- 传感器数据读取
- 执行器控制
- 通信接口驱动（I2C、SPI、UART 等）

**文件命名**：
- `sensor_xxx.c/h` - 传感器驱动
- `display_xxx.c/h` - 显示驱动
- `xxx_bus.c/h` - 总线驱动

### 5. 中间件层 (middleware)
**职责**：
- 数据存储（NVS、Flash、文件系统）
- 加解密（TLS/SSL、AES、RSA）
- 日志系统
- 缓存管理

### 6. 系统服务层 (service)
**职责**：
- OTA 远程升级
- 配置管理
- 看门狗
- 电源管理

### 7. 公共工具层 (common)
**职责**：
- 通用数据结构
- 工具函数
- 算法实现
- 错误码定义

**文件命名**：
- `queue.c/h` - 队列实现
- `list.c/h` - 链表实现
- `utils.c/h` - 通用工具

## 组件命名规范

| 类型 | 命名方式 | 示例 |
|------|----------|------|
| 协议组件 | `protocol_xxx` | `protocol_mqtt`, `protocol_http` |
| 网络组件 | `net_xxx` | `net_wifi`, `net_eth` |
| 驱动组件 | `drv_xxx` | `drv_sensors`, `drv_display` |
| 服务组件 | `svc_xxx` | `svc_ota`, `svc_config` |

## CMakeLists.txt 模板

### 应用层组件
```cmake
idf_component_register(
    SRCS "app_main.c"
    INCLUDE_DIRS "."
    REQUIRES protocol network driver
)
```

### 协议层组件
```cmake
idf_component_register(
    SRCS "mqtt_client.c" "json_parser.c"
    INCLUDE_DIRS "."
    REQUIRES esp_netif mqtt json
)
```

### 网络层组件
```cmake
idf_component_register(
    SRCS "wifi_manager.c"
    INCLUDE_DIRS "."
    REQUIRES esp_netif esp_wifi nvs_flash
)
```

## 依赖方向

```
应用层 → 协议层 → 网络层 → 驱动层 → ESP-IDF HAL
              ↕         ↕
           中间件    服务层
```

**规则**：
- 上层可以依赖下层
- 下层不应依赖上层
- 同层之间按需依赖
- 避免循环依赖
